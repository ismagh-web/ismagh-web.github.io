3.5. Protocolo PPP (Point to Point Protocol)

PPP es un protocolo fundamental en la pila TCP/IP, operando en la Capa 2 (Enlace de Datos). Está diseñado para manejar comunicaciones a través de enlaces punto a punto, superando las limitaciones de protocolos seriales anteriores.

Estructura y Componentes

  • HDLC-like Framing: Define cómo se encapsulan los datos para su transmisión por la línea serial. Cada trama PPP incluye campos para delimitadores, direcciones, control y una comprobación de errores (FCS).
  • LCP (Link Control Protocol): Es responsable de establecer, configurar y probar la conexión de Capa 2. LCP negocia parámetros como la autenticación, el manejo de errores y el tamaño máximo de paquete (MRU).
  • NCP (Network Control Protocol): Es un conjunto de protocolos (como IPCP para IP) que configuran protocolos de Capa 3 para que puedan operar sobre el enlace PPP.

Variantes Importantes: PPPoE

El **PPPoE (PPP over Ethernet)** es crucial. Permite incrustar las tramas PPP dentro de las tramas Ethernet. Esto es usado por muchos ISP para el acceso a Internet (ADSL/Fibra), ya que permite la **autenticación** individualizada de cada cliente antes de otorgarle acceso a la red IP.
